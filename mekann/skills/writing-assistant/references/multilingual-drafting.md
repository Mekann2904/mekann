# Multilingual drafting workflow

日本語の最終文書でも、初稿の論理設計は LLM が得意な言語で行う。

## 原則

- 既定の作業言語は英語。
- 中国語モデルなど、対象モデルが中国語の作文に強い場合は中国語を使ってよい。
- 最終成果物は日本語で仕上げる。
- 内部ドラフトは通常ユーザに見せない。求められた場合だけ併記する。

## 手順

1. 日本語で要件を整理する。
2. 得意言語でアウトラインを作る。
3. 得意言語で初稿を作る。
4. 日本語に翻訳する。
5. 直訳調を除去し、日本語技術文として編集する。
6. 事実、数値、引用、固有名詞に「要確認」がないか確認する。

## 翻訳後に必ず直すこと

- 英語的な長い名詞句を短くする。
- 主語、述語、目的語を明確にする。
- 「This paper」直訳の不自然さを避け、必要に応じて「本論文」「本手法」「筆者ら」を使い分ける。
- 直訳で生じた曖昧な受動態を能動態にする。
- 断定できない箇所を断定しない。
