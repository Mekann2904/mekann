# Multilingual reasoning workflow

日本語の最終文書でも、推論（課題分解、構成計画、証拠の照合、論理整合の確認）は LLM が思考しやすい言語で行い、最終テキストは日本語で直接構成する。得意言語の初稿を翻訳するのではない。

## 原則

- 推論言語の既定は英語。
- 対象モデルが中国語の思考に強い場合は中国語を使ってよい。
- 最終成果物は日本語で仕上げる。
- 内部推論は通常ユーザに見せない。求められた場合だけ併記する。

## 手順

1. 日本語で要件を整理する。
2. 推論言語でアウトライン・論証・証拠の連なりを組み立てる。
3. その推論を土台にして、日本語で本文を直接構成する（推論テキストを逐語翻訳しない）。
4. 日本語技術文として編集し、直訳調を除去する。
5. 事実、数値、引用、固有名詞に「要確認」がないか確認する。

## 日本語出力で必ず直すこと

- 英語的な長い名詞句を短くする。
- 主語、述語、目的語を明確にする。
- 「This paper」直訳の不自然さを避け、必要に応じて「本論文」「本手法」「筆者ら」を使い分ける。
- 直訳で生じた曖昧な受動態を能動態にする。
- 断定できない箇所を断定しない。
