# ドキュメント管理方針

> パンくず: [Home](../../README.md) > [Meta](./) > ドキュメント管理方針

このプロジェクトでは、AIエージェント（pi/loop機能など）による自動的なドキュメント生成と管理を行います。

## ドキュメントのカテゴリ

| カテゴリ | ディレクトリ | 用途 | 生成元 |
|---------|-----------|------|--------|
| **Getting Started** | `01-getting-started/` | 新規ユーザー向け | 手動作成 |
| **User Guide** | `02-user-guide/` | ユーザーガイド | 手動作成 |
| **Developer Guide** | `03-development/` | 開発者向け | 手動作成 |
| **Reference** | `04-reference/` | リファレンス | 手動作成 |
| **Meta** | `05-meta/` | メタドキュメント | 手動/AI生成 |
| **Archives** | `05-meta/archives/` | 古いドキュメント | 移動 |

## AIエージェントによるドキュメント生成

### 使用コマンド

```bash
/loop run --max <回数> --ref <参照パス> <タスク>
```

### 生成フロー

1. **事前調査** - ソースコード・ドキュメントを参照して分析
2. **レポート作成** - 調査結果をマークダウン形式で出力
3. **配置** - 適切なディレクトリに保存
4. **コミット** - 変更をGitに記録

### 出力ファイルの命名規則

- 調査レポート: `*-report.md` （例: `ui-unification-report.md`）
- 設計書: `*-design.md`
- 要約: `*-summary.md`

### 参照リソース

ルートディレクトリ以下の主要なリソースを参照します：

- `.pi/extensions/` - 拡張機能
- `docs/` - 既存ドキュメント
- `.pi/` - pi設定

## ドキュメント更新のルール

### 更新トリガー

- コード変更によるUI/UXの変更時
- 新しい拡張機能の追加時
- 重大なバグ修正後
- ユーザーからのフィードバック受信時

### 更新手順

1. `/loop run` で最新の状態を調査
2. 既存ドキュメントと照合
3. 必要に応じて更新または新規作成
4. チェックリスト確認後コミット

### 引用（Citations）の使用

AIエージェントによる生成時は `--require-citation` を使用し、情報源を明示します：

```markdown
- 調査発見 [R1]
- 設計方針 [R2][R3]
```

## メタデータ標準

### 全ページ必須フィールド

```yaml
---
title: ページタイトル
category: getting-started | user-guide | development | reference | meta
audience: new-user | daily-user | developer | contributor
last_updated: YYYY-MM-DD
tags: []
related: []
---
```

### フィールド説明

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `title` | ✓ | ページタイトル |
| `category` | ✓ | ドキュメントのカテゴリ |
| `audience` | ✓ | 対象読者 |
| `last_updated` | ✓ | 最終更新日 |
| `tags` | - | 関連キーワード |
| `related` | - | 関連ページへのパス |

## ドキュメントの品質管理

### チェックリスト

- [ ] 適切なカテゴリに配置されている
- [ ] ファイル名が命名規則に従っている
- [ ] 参照元が引用されている（AI生成の場合）
- [ ] メタデータが正しく設定されている
- [ ] ドキュメント内のリンクが有効
- [ ] 最新の状態を反映している
- [ ] パンくずリストが表示されている
- [ ] 関連トピックが記載されている

### レビュー

- AIエージェント生成レポート: 生成後に要約を確認
- 手動作成ドキュメント: プルリクエストでレビュー

## 言語戦略

- **メイン言語**: 日本語
- **コードと技術用語**: 英語
- **ディレクトリ名とファイル名**: 英語（kebab-case）
- **翻訳**: 将来的な検討事項

## ナビゲーション標準

### パンくずリスト

各ページの先頭に表示します：

```markdown
> パンくず: [Home](../../README.md) > [カテゴリ](./) > ページ名
```

### 関連トピック

各ページの末尾に記載します：

```markdown
---

## 関連トピック

- [関連ページ1](path/to/page1.md) - 説明
- [関連ページ2](path/to/page2.md) - 説明

## 次のトピック

[ → 次のページ](path/to/next-page.md)
```
