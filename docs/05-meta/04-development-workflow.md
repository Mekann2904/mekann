# AIエージェントによるドキュメント管理

> パンくず: [Home](../../README.md) > [Meta](./) > AIエージェントによるドキュメント管理

このプロジェクトでは、AIエージェントによる自動的なドキュメント生成と管理を行います。

同時に、品質ループも運用します。

mekann では、実装だけでなく、計画、検証、継続の 3 つの loop を管理対象にします。

## ドキュメント管理の方針

### 品質ループの前提

- 実行ループ: `plan -> edit -> test/build/lint -> observe -> repair -> repeat`
- 検証ループ: tests、lint、型検査、必要なら browser / review
- 継続ループ: live todo、`plans/*.md`、進捗ログ、checkpoint

ドキュメントもこの loop の一部です。

判断理由、verify 手順、proof artifacts を残せない変更は、長期運用で弱くなります。

### ドキュメントのカテゴリ

| カテゴリ | 場所 | 用途 | 生成元 |
|---------|------|------|--------|
| **拡張機能開発** | `docs/03-development/` | 拡張機能開発ガイド | 手動作成 |
| **公式ドキュメント** | `docs/` | 拡張機能の公式ドキュメント | 手動作成 |

### AIエージェントによるドキュメント生成

#### 使用コマンド

```
/loop run --max <回数> --ref <参照パス> <タスク>
```

#### 生成フロー

1. **事前調査** - ソースコード・ドキュメントを参照して分析
2. **レポート作成** - 調査結果をマークダウン形式で出力
3. **検証計画** - 何を verify するか、何を証拠として残すか決める
4. **配置** - 適切なディレクトリに保存
5. **更新記録** - plan や progress log に現在地を残す
6. **コミット** - 変更をGitに記録

#### 出力ファイルの命名規則

- 調査レポート: `*-report.md` （例: `ui-unification-report.md`）
- 設計書: `*-design.md`
- 要約: `*-summary.md`

#### 参照リソース

ルートディレクトリ以下の主要なリソースを参照します：

- `extensions/` - ローカル拡張機能
- `.pi/extensions/` - pi組み込み拡張機能
- `docs/` - 既存ドキュメント
- `.pi/examples/` - 公式サンプルコード

## ドキュメント更新のルール

### 更新トリガー

- コード変更によるUI/UXの変更時
- 新しい拡張機能の追加時
- 重大なバグ修正後
- ユーザーからのフィードバック受信時

### 更新手順

1. `/loop run` で最新の状態を調査
2. 既存ドキュメントと照合
3. 必要に応じて更新または新規作成
4. verify 手順と proof artifacts を追記
5. チェックリスト確認後コミット

### 引用（Citations）の使用

AIエージェントによる生成時は `--require-citation` を使用し、情報源を明示します：

```markdown
- 調査発見 [R1]
- 設計方針 [R2][R3]
```

## ドキュメントの品質管理

### チェックリスト

- [ ] 適切なカテゴリに配置されている
- [ ] ファイル名が命名規則に従っている
- [ ] 参照元が引用されている（AI生成の場合）
- [ ] ドキュメント内のリンクが有効
- [ ] 最新の状態を反映している
- [ ] 実行した verify 手順が残っている
- [ ] proof artifacts または未検証理由が残っている

### レビュー

- AIエージェント生成レポート: 生成後に要約を確認
- 手動作成ドキュメント: プルリクエストでレビュー
