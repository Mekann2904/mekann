# codex-shared を先行抽出する

`codex-web-search` の追加にあたり、共通する Codex API クライアント層（URL 正規化、認証ヘッダ生成、accountId 抽出、エラー型、モデル一覧取得・キャッシュ）を `codex-shared` モジュールとして先行抽出した。代替案は「各モジュールに複製し、後で共有化する」だった。先行抽出を選んだ理由は、重複対象が既に 6 つ以上具体化しており、複製後にズレた場合の認証・エラー分類・モデル解決の不整合リスクが高いため。後からの共有化は動いている両モジュールを同時に再触るリスクを増す。`codex-shared` は Pi フレームワーク型に依存せず、純粋な Codex HTTP クライアント基盤に留める。
