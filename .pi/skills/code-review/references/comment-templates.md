# コードレビューコメントテンプレート

## Nit（あら探し・細かい指摘）

必須ではないが改善を提案する場合に使用。

```
Nit: [指摘内容]
```

**例:**
```
Nit: この変数名は `userCount` の方が説明的かもしれませんが、必須ではありません。
```

---

## 設計に関するコメント

### 複雑すぎる設計

```
この[関数/クラス/モジュール]は複雑すぎるように見えます。
[具体的な理由]
[提案する代替案]
```

### オーバーエンジニアリング

```
この実装は[現在の要件]に対して過度に一般化されているように見えます。
現在解決する必要のある問題に集中し、将来の要件は実際に発生してから対応することをお勧めします。
```

---

## 機能性に関するコメント

### エッジケース

```
[エッジケースの説明]の場合、このコードはどう動作しますか？
テストを追加して確認することをお勧めします。
```

### 並行処理の問題

```
ここでの並行処理には潜在的な問題がある可能性があります。
[デッドロック/競合状態]が発生するシナリオ：
[説明]
```

---

## テストに関するコメント

```
この変更にはテストが不足しています。
以下のシナリオをカバーするテストを追加してください：
- [シナリオ1]
- [シナリオ2]
```

---

## 可読性に関するコメント

### 命名

```
この[変数/関数]の名前は[理由]で誤解を招く可能性があります。
[提案する名前]はより明確ではないでしょうか？
```

### コメント

```
このコメントは「何」を説明していますが、「なぜ」このアプローチを選択したのかを説明すると、将来の保守に役立ちます。
```

---

## セキュリティに関するコメント

```
セキュリティ上の懸念があります。
[問題点]
[推奨される対処]
```

---

## 肯定的なフィードバック

```
素晴らしい実装です！[具体的に良い点]が特にうまくできています。
```
