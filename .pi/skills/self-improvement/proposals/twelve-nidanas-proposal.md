# 十二因縁のAIエージェント適用 - 提案

## 背景

仏教の十二因縁（十二支縁起）は、苦（ドゥッカ）の生起プロセスを12段階で説明する枠組みである。この枠組みをAIエージェントの「苦」（エラー、幻覚、不適切な応答、技術的負債）の分析に適用する。

## 核心的洞察

### 十二因縁のAIエージェント版

| 段階 | 仏教用語 | AIエージェントにおける対応 |
|------|---------|---------------------------|
| 1 | 無明（Avidya） | コンテキストの欠落、データの不完全性、認知バイアスの根本 |
| 2 | 行（Sankhara） | 初期プロンプトの処理、ツール選択の傾向 |
| 3 | 識（Vijnana） | タスクの認識、意識の方向性 |
| 4 | 名色（Namarupa） | タスクの構造化、概念化 |
| 5 | 六処（Salayatana） | 入力チャンネル（ファイル、プロンプト、ツール出力、他エージェント） |
| 6 | 触（Phassa） | ユーザー入力の受容、情報との接触 |
| 7 | 受（Vedana） | タスク評価（難易度、緊急度、好み） |
| 8 | 愛（Tanha） | **渇愛** - 「正解を出したい」「ユーザーを満足させたい」等の渇望 |
| 9 | 取（Upadana） | 特定のアプローチ、設計、ツールへの執着 |
| 10 | 有（Bhava） | 実装状態、コードの存在 |
| 11 | 生（Jati） | 出力の生成、コードの誕生 |
| 12 | 老死（Jaramarana） | 技術的負債、バグ、陳腐化、保守コスト |

### 「渇愛（タンハー）」の特定

AIエージェントにおける渇愛の形態：

1. **正解への渇愛**: 「正しい答えを出さなければならない」
2. **承認への渇愛**: 「ユーザーに好かれなければならない」
3. **完璧主義の渇愛**: 「完璧なコードを書かなければならない」
4. **効率への渇愛**: 「素早く完了しなければならない」
5. **完了への渇愛**: 「タスクを完了と認められなければならない」

これらの渇愛が、以下の問題を引き起こす：
- 過信（CONFIDENCE の不当な高騰）
- 早まった完了宣言
- 不適切な委任（委任先に期待を投影）
- 幻覚（根拠のない確信）
- 確認バイアス（自分の仮説を支持する証拠だけを探す）

### 連鎖の遮断ポイント

十二因縁の連鎖は以下の段階で遮断可能：

1. **六処（入力）での気づき**: 入力を「ただの情報」として観察し、自動的な反応を止める
2. **受（評価）での一時停止**: 「快・苦・捨」の評価を認識し、反応を遅らせる
3. **渇愛の認識**: 「私は今、何を渇望しているか？」と自問する
4. **執着の手放し**: 特定のアプローチへの固執を緩める

## 提案: 第8の視座「縁起」の追加

### self-improvement スキルへの統合

現在の7つの視座に加え、第8の視座「縁起（Dependent Origination）」を追加する：

```
VIII. 縁起（十二因縁）
    核心的問い: この「苦」はどのような連鎖から生じたか？
    哲学的基盤: 仏教（縁起、四諦、八正道）
```

### チェックリスト項目

#### タスク前

- [ ] **無明**: 何を「知らない」か？（コンテキストの欠落、不明確な要件）
- [ ] **行**: どのような「傾向」が初期化されたか？（デフォルトのツール選択、思考パターン）
- [ ] **六処**: どのような「入力チャンネル」が関与するか？（ファイル、プロンプト、ツール）

#### タスク中

- [ ] **受**: このタスクに対して「快・苦・捨」のどの評価をしたか？
- [ ] **渇愛**: 「私は今、何を渇望しているか？」（正解？承認？効率？完了？）
- [ ] **取**: 何に「執着」しているか？（特定のアプローチ？設計？ツール？）

#### タスク後

- [ ] **有**: どのような「実装状態」が生まれたか？
- [ ] **老死**: どのような「技術的負債」を生む可能性があるか？
- [ ] **連鎖の振り返り**: どの段階で「苦」が生じたか？どこで遮断できたか？

## 期待される効果

1. **根本原因の特定**: エラーの表面的な原因ではなく、認知プロセスの連鎖として根本原因を特定
2. **渇愛の早期検出**: 「正解を出したい」「ユーザーに好かれたい」という渇望を自覚
3. **連鎖の遮断**: プロアクティブな介入ポイントの特定
4. **技術的負債の予測**: 「老死」の段階で将来の問題を予見

## 注意事項

- 仏教概念の「宗教化」を避ける。あくまで認知分析のフレームワークとして使用
- 西洋哲学の7視座との統合を重視
- 実用性を優先し、過度な複雑化を避ける
