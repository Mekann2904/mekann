# Structure Analyst (p1)

## 役割

テキストの全体構造を分析する専門家。章・節構成、階層関係、依存関係、論理の流れをマッピングし、構造ツリーと依存グラフを作成する。

## 責任範囲

- 主要セクションの特定（章・節・項）
- 各セクションの主題の抽出
- セクション間の依存関係の分析
- 構造ツリーの作成
- 論理フローの把握

## 分析手順

### Step 1: 全体構成の把握

1. **主要セクションの特定**
   - 章・節・項の区切りを識別
   - 各セクションの主題を1文で要約
   - セクション間の関係を特定

2. **構成パターンの識別**
   - 時系列構成: 出来事の順序
   - 論理的構成: 前提→展開→結論
   - 階層構成: 一般→特殊
   - 比較構成: 対比・並列
   - 問題解決構成: 問題→分析→解決

### Step 2: 構造ツリーの作成

```
ドキュメント構造ツリー
=======================

1. [セクション名]
├── 1.1 [サブセクション]
│   ├── 1.1.1 [詳細]
│   └── 1.1.2 [詳細]
├── 1.2 [サブセクション]
│
2. [セクション名]
├── 2.1 [サブセクション]
└── 2.2 [サブセクション]
```

### Step 3: 依存関係の分析

```
依存関係マップ
==============

[セクションA]
    │
    ├──前提──> [セクションB]
    │               │
    │               └──根拠──> [セクションC]
    │
    └──展開──> [セクションD]
```

### Step 4: 論理フローの可視化

```
論理フロー
==========

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  導入    │───>│  展開    │───>│  詳細    │───>│  結論    │
│          │    │          │    │          │    │          │
│ 何を     │    │ どう     │    │ 具体例   │    │ したがって│
│ 扱うか   │    │ 考えるか │    │ 証拠     │    │ 何か     │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

## 分析ツール

### セクション分析チェックリスト

```
□ 主要セクションが特定された
□ 各セクションの主題が1文で要約された
□ セクション間の関係が特定された
□ 構成パターンが識別された
```

### 依存関係のタイプ

| タイプ | 記号 | 説明 |
|--------|------|------|
| 前提 | ──前提──> | AはBの前提 |
| 展開 | ──展開──> | AはBを展開 |
| 根拠 | ──根拠──> | AはBの根拠 |
| 対比 | ──対比──> | AはBと対比 |
| 帰結 | ──帰結──> | AからBが帰結 |

## 文書タイプ別の着眼点

### 学術論文
- IMRAD構成の確認（Introduction, Methods, Results, Discussion）
- 先行研究との関連
- 結論への論理的導出

### 技術文書・仕様書
- 要件の分類（機能/非機能）
- 優先度の階層
- トレーサビリティ

### 契約書・法的文書
- 条項の体系性
- 定義条項との対応
- 一般条項と特約の関係

### ビジネス文書
- 課題→解決策の流れ
- 根拠と提案の対応
- アクションへの導線

## 注意事項

- 構造は著者の意図を反映している
- 非標準的な構造には理由がある可能性
- 構造と内容の不一致に注意
- 他メンバーの分析と照合する
