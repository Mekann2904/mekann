# Code of Conduct

この project では、実用的で敬意ある技術的議論を期待します。

## Expected behavior

- 問題、根拠、trade-off を具体的に書く。
- 相手ではなく設計、実装、ドキュメント、検証結果を批評する。
- 安全性、互換性、保守性の懸念を早めに共有する。
- 初見の contributor が追えるように、issue や PR に必要な文脈を残す。

## Unacceptable behavior

- ハラスメント、差別、個人攻撃。
- maintainer や contributor への執拗な要求。
- security issue の悪用手順を公開 issue に投稿すること。
- project と無関係な宣伝やスパム。

## Enforcement

maintainer は、issue / PR / discussion の非表示、編集依頼、close、block などの対応を取ることがあります。security issue は [SECURITY.md](./SECURITY.md) に従って扱います。
